# 미스터 대박 저녁 배달 시스템 요구명세서

## 1. 개요

### 1.1 목적

본 문서는 **‘미스터 대박’ 저녁 배달 서비스**의 원활한 운영을 지원하기 위한 소프트웨어 시스템의 요구사항을 정의한다. 본 시스템은 고객의 주문 편의성을 높이고, 직원의 효율적인 주문 처리 및 배달 관리를 가능하게 하여 비즈니스 성장을 지원한다. 또한 데이터 수집을 통해 고객 주문 패턴을 관리하고, 재고 관리 및 메뉴 개발에 활용하여 **비즈니스 경쟁력 강화**를 목표로 한다.

### 1.2 범위

본 시스템은 **고객용 인터페이스**와 **직원용 인터페이스**로 구성되며, 직원과 고객으로 분류된 로그인 페이지를 통해 서로 다른 인터페이스를 제공한다.

* **고객용 인터페이스**: 웹/앱에서 메뉴 확인, 디너 주문, 결제 기능 제공. GUI 및 **음성 인식** 주문 모두 지원.
* **직원용 인터페이스**: 접수 주문 확인, 주방 준비, 배달 관리 기능 제공.

## 2. 비즈니스 요구사항

### 2.1 비즈니스 목표

* 특별한 날을 위한 **고급 디너**를 고객의 집으로 편리하게 배달하여 최상의 고객 경험 제공
* **유연한 주문 옵션**(메뉴 변경/추가/삭제)으로 고객 만족도 제고
* **단골 고객 확보**를 통한 안정적 수익 모델 구축

### 2.2 메뉴 및 스타일 정책

**디너 종류**

* **발렌타인 디너**: 와인, 스테이크, 하트/큐피드 장식
* **프렌치 디너**: 커피, 와인, 샐러드, 스테이크
* **잉글리시 디너**: 에그 스크램블, 베이컨, 빵, 스테이크
* **샴페인 축제 디너 (2인)**: 샴페인 1병, 바게트 4개, 커피 1포트, 와인, 스테이크

**메뉴 선택**

* 기본 **케이크 추가 주문** 가능
* 업장별 **메뉴 차별화**가 가능하도록 인터페이스 제공 *(추가 사항)*

**서빙 스타일**

* **심플 (Simple)**: 플라스틱 식기, 종이 냅킨, 플라스틱 쟁반
* **그랜드 (Grand)**: 도자기 식기, 면 냅킨, 나무 쟁반 *(와인잔: 플라스틱)*
* **디럭스 (Deluxe)**: 꽃병, 도자기 식기, 린넨 냅킨, 나무 쟁반 *(와인잔: 유리)*

**비즈니스 규칙**

* 디너 **가격은 서빙 스타일에 따라 차등 적용**
* **샴페인 축제 디너**는 **그랜드/디럭스** 스타일로만 주문 가능
* 고객은 주문 과정에서 **추가/수량 변경/삭제** 가능
* **케이크 카드/케이크 아트** 이미지는 **생성형 AI** 기반 생성/업로드 → 프린팅 제공으로 고객 경험 극대화 *(추가 사항)*

## 3. 소프트웨어 요구사항

### 3.1 기능 요구사항

#### 3.1.1 사용자 관리

* **FR-001 (회원가입)**: 성명/주소/전화번호/비밀번호 입력으로 가입, **개인정보 활용 동의** 필수
* **FR-002 (로그인/로그아웃)**: ID(연락처 또는 이메일) + 비밀번호로 인증
* **FR-003 (비밀번호 재설정)**: 가입 정보 기반 비밀번호 재설정 기능 제공
* **FR-004 (고객 정보 저장)**: 동의 고객의 성명/주소/연락처를 **안전하게 저장**
* **FR-005 (직원 정보 저장)**: 직원 **등급별 월급/직급/권한 관리**, 매니저(사장)는 직원 정보·직급·월급·업무 변경 수행 가능. **사장 권한/아이디는 본사 관리**
* **FR-006 (주문 명세 저장)**: 모든 주문을 DB에 저장. **10회 주문마다 이벤트**—VIP 등급 고객에게 사이드디쉬(예: 봉골레 파스타) 자동 추가(업장별 변환 가능)
* **FR-007 (메뉴 재고 관리 시스템)**: 회계·재고·입출금 내역 확인, 온라인 식자재 장부, **푸드 코스트 통계**로 손익 관리

#### 3.1.2 고객 주문 인터페이스

* **FR-008 (주문 인터페이스 제공)**: **GUI + 음성 인식** 2가지 주문 방식 지원
* **FR-009 (메뉴 조회)**: 디너 메뉴/서빙 스타일의 **구성·가격 상세 정보** 제공
* **FR-010 (주문 생성)**: 디너/스타일/수량 선택으로 주문 생성
* **FR-011 (주문 맞춤화)**: 항목 **수량 변경/추가/삭제** 지원
* **FR-012 (배달 정보 입력)**: **배달 장소·시간 지정**, 운송 중 **예상 배달시간 산정/표시**
* **FR-013 (결제)**: **신용카드 결제** 지원
* **FR-014 (과거 주문 목록 조회)**: 로그인 고객의 과거 주문 **최신순 조회**—주문 시간, 디너 내용, 가격, 배달 시간, 배달 주소 포함
* **FR-015 (간편 주문)**: 과거 주문을 **원클릭 재주문**

#### 3.1.3 직원 인터페이스

* **FR-016 (주문 접수 확인)**: 실시간 유입 주문 확인·접수
* **FR-017 (주문 처리 상태 관리)**: ‘주문 접수/조리 중/배달 중/배달 완료’ **상태 변경/관리**
* **FR-018 (주문 상세 정보 조회)**: 디너 종류, 스타일, 맞춤 옵션, 고객/배달지 등 **주문 상세 확인**

#### 3.1.4 할인 기능

* **FR-019 (단골 고객 할인)**: 주문 횟수·금액 등 **조건 충족 시 자동 할인**

#### 3.1.5 부가 기능

* **FR-020 (ASR 활용 챗봇 주문)**: **음성인식(ASR)** 챗봇으로 메뉴 추천·주문·결제 안내 상호작용
* **FR-021 (커스텀 데코레이션 요청)**: 케이크 등 특정 메뉴에 **이미지 업로드/생성**, **문구 입력** 기능 제공

### 3.2 비기능 요구사항

**NFR-001 (보안)**

* 개인정보·신용카드 정보는 **암호화 저장/전송**
* 외부 **비인가 접근 차단**

**NFR-002 (사용성)**

* **직관적 UI**로 최소 단계 주문 완결
* 음성 인식 **높은 정확도**, 발화 처리 명확

**NFR-003 (성능)**

* 다수 동시 접속 상황에서도 **3초 이내 응답**

**NFR-004 (안정성)**

* **24/7 운영**, 장애 시 **신속 복구**

**NFR-005 (시스템 연동성)**

* ASR 챗봇, 이미지 처리 등 핵심 기능에 **외부 전문 API 연동** 및 안정성 보장

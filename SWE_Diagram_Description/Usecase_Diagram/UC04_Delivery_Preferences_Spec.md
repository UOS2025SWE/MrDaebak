# UC04: Manage Delivery Preferences

## 유즈케이스 ID
**MRD-CUS-UC04**

## 액터
- **주 액터**: Customer
- **부 액터**: 없음

## 요약
고객이 기본 배달 정보(이름, 전화번호, 주소)를 조회하고 수정하여 향후 주문 시 자동으로 사용되도록 설정한다.

## 유형
**Secondary**

## 참고 유즈케이스
- **Include**: 없음
- **Extend**: 없음

## 사전 조건
고객이 로그인되어 있다

## 사후 조건
(성공 시) 배달 정보가 저장되고 다음 주문 시 자동으로 적용된다, (실패 시) 오류 메시지가 표시되고 기존 정보가 유지된다

## 주흐름

1. 고객이 배달 정보 조회를 시스템에 요청한다
2. 시스템이 저장된 기본 배달 정보(이름, 전화번호, 주소)를 반환한다
3. 고객이 수정한 배달 정보(이름, 전화번호, 주소)를 시스템에 제출한다
4. 시스템이 배달 정보를 저장하고 저장 완료 메시지를 반환한다

## 부흐름

### 2a. 저장된 배달 정보 없음
(3) 시스템이 고객의 기본 배달 정보가 없음을 감지한다.
(4) 시스템이 빈 폼을 표시하고 "배달 정보를 입력해 주세요." 메시지를 표시한다.
흐름 (3)으로 복귀한다.

### 3a. 주소 형식 오류
(5) 시스템이 입력된 주소 형식이 유효하지 않음을 감지한다.
(6) 시스템이 "올바른 주소를 입력해 주세요." 오류 메시지를 표시한다.
흐름 (3)으로 복귀한다.

### 3b. 전화번호 형식 오류
(5) 시스템이 입력된 전화번호 형식이 유효하지 않음을 감지한다.
(6) 시스템이 "올바른 전화번호를 입력해 주세요." 오류 메시지를 표시한다.
흐름 (3)으로 복귀한다.

## 제약사항
- 전화번호는 숫자만 입력 가능하며 10-11자리여야 한다.
- 주소는 배달 가능 지역 내에 있어야 하며, 상세 주소를 포함해야 한다.

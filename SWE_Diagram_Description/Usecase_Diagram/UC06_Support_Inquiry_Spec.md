# UC06: Submit Support Inquiry

## 유즈케이스 ID
**MRD-CUS-UC06**

## 액터
- **주 액터**: Customer
- **부 액터**: 없음

## 요약
고객이 서비스 이용 중 문의 사항이나 문제를 시스템에 제출하여 지원을 요청한다.

## 유형
**Secondary**

## 참고 유즈케이스
- **Include**: 없음
- **Extend**: 없음

## 사전 조건
고객이 시스템에 접근 가능하다 (로그인 선택 사항)

## 사후 조건
(성공 시) 문의가 저장되고 문의 ID와 상태가 반환된다, (실패 시) 오류 메시지가 표시되고 문의가 저장되지 않는다

## 주흐름

1. 고객이 지원 문의 정보(이름, 이메일, 주제, 메시지)를 시스템에 제출한다
2. 시스템이 문의를 저장하고 문의 접수 확인(문의 ID, 상태)을 반환한다

## 부흐름

### 1a. 필수 정보 누락
(3) 시스템이 필수 정보(이름, 이메일, 주제, 메시지)가 누락되었음을 감지한다.
(4) 시스템이 "모든 필수 정보를 입력해 주세요." 오류 메시지를 표시한다.
흐름 (1)로 복귀한다.

### 1b. 이메일 형식 오류
(3) 시스템이 입력된 이메일 형식이 유효하지 않음을 감지한다.
(4) 시스템이 "올바른 이메일 주소를 입력해 주세요." 오류 메시지를 표시한다.
흐름 (1)로 복귀한다.

### 2a. 서버 오류
(3) 시스템에서 데이터베이스 저장 오류가 발생한다.
(4) 시스템이 "일시적인 문제가 발생했습니다. 잠시 후 다시 시도해 주세요." 메시지를 표시한다.
흐름 (1)로 복귀한다.

## 제약사항
- 문의 메시지는 최소 10자 이상이어야 한다.
- 등록된 문의는 관리자가 검토할 때까지 "NEW" 상태로 유지되며, 고객은 문의 ID로 상태를 추적할 수 없다.

전체 시스템은 아래의 서브 시스템으로 구성된다.
-    Customer Web App 시스템
-    Staff Web App 시스템
-    Admin Web App 시스템
-    Order & Checkout Domain 시스템
-    Menu & Inventory Domain 시스템
-    Event & Discount Domain 시스템
-    User & Auth Domain 시스템
-    Staff Management Domain 시스템
-    AI & Realtime Domain 시스템
-    Database Layer 시스템
-    External Systems 시스템

Customer Web App 시스템
-    고객이 메뉴를 조회하고, 주문하며, 배달 상태를 추적하는 웹 애플리케이션 역할을 한다.
-    GUI 방식 및 음성 주문을 통해 주문을 진행할 수 있으며, 실시간으로 주문 상태를 확인한다.
-    Next.js 15, React 18, TypeScript 언어를 사용하여 작성된다.

Staff Web App 시스템
-    요리사와 배달원이 배정된 주문을 확인하고 상태를 업데이트하는 웹 애플리케이션 역할을 한다.
-    실시간으로 새로운 주문 배정 알림을 받고, 주문 진행 상황에 따라 상태를 변경한다.
-    Next.js 15, React 18, TypeScript 언어를 사용하여 작성된다.

Admin Web App 시스템
-    매니저가 시스템 전반을 관리하고 모니터링하는 웹 애플리케이션 역할을 한다.
-    통계 대시보드를 통해 시스템 현황을 파악하고, 메뉴, 재료, 이벤트, 직원 등을 종합적으로 관리한다.
-    Next.js 15, React 18, TypeScript 언어를 사용하여 작성된다.

Order & Checkout Domain 시스템
-    주문 생성, 조회, 상태 변경 등 핵심 주문 로직을 처리하는 역할을 한다.
-    결제 처리 및 커스텀 케이크 주문을 담당하며, 필요시 External Systems와 통신한다.
-    FastAPI, Python 3.12 언어를 사용하여 작성된다.

Menu & Inventory Domain 시스템
-    메뉴 정보, 가격, 실시간 재고를 관리하는 역할을 한다.
-    주문 발생 시 재고를 차감하고, 매니저와 요리사가 협력하는 2단계 입고 프로세스를 통해 재고를 관리한다.
-    FastAPI, Python 3.12 언어를 사용하여 작성된다.

Event & Discount Domain 시스템
-    프로모션 이벤트와 고객 충성도 할인을 관리하는 역할을 한다.
-    고객의 주문 이력을 기반으로 충성도 등급 할인을 자동 적용하며, 할인은 충성도, 이벤트 순서로 누적 계산된다.
-    FastAPI, Python 3.12 언어를 사용하여 작성된다.

User & Auth Domain 시스템
-    JWT 기반 인증과 고객 지원 문의를 관리하는 역할을 한다.
-    고객 및 직원의 회원가입, 로그인, 프로필 관리를 담당하며, 고객 문의 등록 및 상태 관리 기능을 제공한다.
-    FastAPI, Python 3.12 언어를 사용하여 작성된다.

Staff Management Domain 시스템
-    직원 포지션 배정과 성과 추적을 관리하는 역할을 한다.
-    신규 직원의 포지션을 배정하고, 직원 근무 상태 및 성과를 추적하며, 직원 해고 처리 및 사유 기록 기능을 제공한다.
-    FastAPI, Python 3.12 언어를 사용하여 작성된다.

AI & Realtime Domain 시스템
-    AI 음성 주문과 실시간 WebSocket 알림을 처리하는 역할을 한다.
-    Google Gemini를 사용한 음성 주문 자연어 이해, OpenAI Audio API를 통한 STT/TTS 변환, 대화 세션 관리 및 컨텍스트 유지 기능을 제공한다.
-    FastAPI, Python 3.12, Google Gemini, OpenAI Audio 언어 및 API를 사용하여 작성된다.

Database Layer 시스템
-    시스템의 모든 영구 데이터를 저장하는 중앙 데이터베이스 역할을 한다.
-    User, Order, Product 등 각 스키마로 데이터를 구조화하여 저장하고, 백엔드 서비스의 SQL Query에 응답한다.
-    PostgreSQL 17, SQLAlchemy ORM을 사용하여 작성된다.

External Systems 시스템
-    외부 API 및 서비스와의 통합을 담당하는 역할을 한다.
-    AI 기반 기능 제공(Gemini API), 음성 변환(OpenAI Audio API), 결제 처리(Mock Payment Gateway), 파일 저장(S3), 배달 경로 조회(Maps API) 등을 처리한다.
-    Google Cloud AI, OpenAI, Mock Payment Gateway 등의 External API를 사용한다.

아키텍처도에서 서브 시스템간의 실선은 다음의 경우를 의미한다.
-    시작 서브 시스템에서 대상 서브 시스템으로 명확한 API 요청/응답 메시지가 전달되는 경우
-    시작 서브 시스템이 대상 서브 시스템의 데이터베이스 스키마에 직접적인 CRUD(SQL Query) 의존성을 가지는 경우

아키텍처도에서 서브 시스템간의 점선은 다음의 경우를 의미한다.
-    직접적인 요청 흐름을 우회하는 보조적인 메시지가 전달되는 경우
-    주요 데이터베이스 스키마 외에, 다른 스키마의 데이터를 참조(JOIN 등)하는 경우
